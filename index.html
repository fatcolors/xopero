Slajd 03

Jak zwiększyć inne wskaźniki?

Scroll depth 18% → 45%

Użytkownicy z płatnych reklam (Google Ads / LinkedIn Ads) mają bardzo mało cierpliwości. Klikają, bo mają konkretny problem (np. szukają backupu po incydencie), ale możliwe jest też, że po prostu zaciekawiła ich reklama.

Jeśli w ciągu 3 do 5 sekund nie upewnią się, że są we właściwym miejscu i nie poczują zaufania opuszczają stronę.

Views per active user 1.2 → 3.1

Obecnie strona to długi landing page. Jeśli chcemy zwiększyć liczbę odsłon na użytkownika, musimy inteligentnie linkować do innych zasobów na wcześniejszych etapach ścieżki.

Np. cennik, szczegółowe use case'y, security page, case studies, a nie tylko wymuszać konwersję na Trial.

Slajd 04

Hero jest efektowny, ale nie odpowiada szybko na najważniejsze pytania

Pierwsza sekcja pod Hero to „Time-savvy enterprise GitHub backups". Grafika jest estetyczna, ale komunikat jest dość standardowy. „Robimy zautomatyzowane backupy". Sformułowanie nie rozwiązuje żadnego problemu. Brzmi jak slogan, a nie obietnica wartości. Opis też jest zdecydowanie za długi. Użytkownik z reklamy skanuje stronę w 3 sekundy.

Dyrektora z branży Healthcare / FinTech interesuje bezpieczeństwo, zgodność z prawem i uniknięcie katastrofy. Tu zaczynamy od funkcji, zamiast od zbudowania zaufania lub uświadomienia ryzyka.

Slajd 05

Hipotezy i propozycje optymalizacji

Zwiększenie zaufania natychmiast po wejściu na stronę zwiększy scroll depth.

Optymalizacja

Przeniesienie logo „Trusted by…" (najlepiej logo od Healthcare & MedTech) oraz małe badge G2 High Performer bezpośrednio pod sekcję Hero zanim użytkownik zacznie scrollować. Dodatkowo skrócenie tekstu oraz przeredagowanie go na wzbudzenie strachu, propozycję wartości

Dlaczego to zadziała

Osoby z branż regulowanych od razu widzą, że inni „więksi" już wam zaufali. Dodanie Social proof, zwiększenie widoczności CTA oraz dodanie informacji o prostocie założenia konta będzie mieć pozytywny wpływ na konwersję

Slajd 06

Edukacja o „ryzyku" na wczesnym etapie

…zatrzyma uwagę i zachęci do czytania (wzrost scroll depth).

Optymalizacja

Przeniesienie ciemnej sekcji „Still think your data is already backed up?" z cytatem z GitHub ToS znacznie wyżej. Przeprojektowanie na bardziej interaktywny moduł np. zakładki z ToS GitHuba, GitLaba, Bitbucketa. Dodatkowo kilka zmian wizualnych. Kolor secondary text nie jest spójny z hero. Text na buttonie nie przechodzi walidacji kontrastu (AA) dla normalnego tekstu.

Dlaczego to zadziała

W pierwszej kolejności uderzamy w ból użytkownika. Uświadamiamy mu, że jego natywne narzędzia nie dają bezpieczeństwa. To wywołuje zaciekawienie i chęć poznania rozwiązania. Dodatkowo ujednolicenie komponentów oraz drobne zmiany wizualne jak przeniesienie ikony (strzałki) z Secondary buttona do Primary Buttona pozytywnie wpłynie na konwersję sugerując, który button należy wcisnąć.

Slajd 07

Poprawa dostępności przycisku CTA

Optymalizacja

Dodanie dodatkowej sekcji CTA w mega menu. Dodanie małymi literami pod przyciskiem CTA dodatkowych informacji zachęcających do kliknięcia.

Dlaczego to zadziała

Użytkownik może chcieć skonwertować w dowolnym momencie czytania. Sticky header sprawia, że nie musi szukać przycisku. Microcopy uspokaja jego obawy.

Slajd 10

Co można zrobić dodatkowo?

Skrócenie i przegrupowanie środkowej części strony. Teraz środek strony sprawia wrażenie długiej listy benefitów o podobnej wadze. Zamiast wielu podobnych bloków 3 większe grupy:

What GitProtect protects

Repozytoria, metadata, konfiguracja, inne krytyczne obiekty

How recovery works

Restore granularny, DR, szybkość odzyskania, scenariusze awarii

Why enterprise teams trust it

Security, compliance, residency, deployment, auditability

Każdy scroll przynosi nową kategorię wiedzy, a nie „kolejny podobny feature".

Slajd 11

Przeniesienie FAQ dużo wyżej

FAQ na końcu strony jest za późno. Po pierwszej lub drugiej sekcji dodać "mini FAQ" a na końcu strony zostawić pełne FAQ.

What GitHub data is covered? How fast can I restore? Where is backup data stored? Is it compliant for regulated industries? Do I need admin access? Can I try it before talking to sales?
Dlaczego to zadziała

W enterprise użytkownik nie klika trial tylko dlatego, że „produkt wygląda dobrze". Kliknie, kiedy poczuje, że najważniejsze ryzyka zostały wyjaśnione.

Slajd 12

Poprawa message match

Zamiast jednej uniwersalnej wersji landing page'a, przygotować 2 do 3 warianty w tym samym systemie:

GitHub Backup for Healthcare

GitHub Disaster Recovery for DevOps leaders

GitHub Security & Compliance for CISO / CTO

Nie trzeba robić nowego designu. Wystarczy podmiana:

  • headline
  • subheadline
  • pierwszych proof pointów
  • case study
  • CTA context
Slajd 13

Poprawa spójności komponentów

Problemy

  • Dużo kart o podobnej funkcji, ale różnej „wadze" wizualnej
  • Nierówna hierarchia nagłówków
  • Miejscami duże puste przestrzenie między blokami
  • Podobny pattern sekcji powtarzany zbyt długo
  • CTA powinny mieć bardziej konsekwentną hierarchię

Rozwiązania

  • Jeden system spacingu sekcji
  • Jeden system kart feature / proof
  • Jeden primary CTA style i jeden secondary
  • Konsekwentna typografia H1 / H2 / H3
  • Mniejsze rozproszenie przez dekoracje
